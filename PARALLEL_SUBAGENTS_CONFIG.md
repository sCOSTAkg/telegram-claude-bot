# ⚡ Конфигурация параллельных субагентов

## 📊 Текущий статус
✅ **ВКЛЮЧЕНО ПО-УМОЛЧАНИЮ** для всех пользователей

```javascript
// bot.js строка 1468
parallelSubAgents: true
```

## 🎯 Как работает

### На шаге 1 (анализ запроса):
1. **Определение сложности** → complexity: simple|medium|complex|very_complex
2. **Выбор яруса моделей** → PARALLEL_SUBAGENT_TIERS[complexity]
3. **Параллельный запуск** → все модели работают ОДНОВРЕМЕННОоднов
4. **Селектор** → Gemini 2.0 flash-lite выбирает ЛУЧШИЙ ответ
5. **Fallback** → если селектор упал, берётся самый длинный ответ

### Ярусы сложности (bot.js строка 9129):

```javascript
PARALLEL_SUBAGENT_TIERS = {
  simple:        ['gemini-2.5-flash', 'claude-haiku', 'gpt-4.1-mini'],
  medium:        ['claude-sonnet', 'gemini-2.5-flash', 'gpt-4.1'],
  complex:       ['claude-sonnet', 'gemini-2.5-pro', 'gpt-4.1'],
  very_complex:  ['claude-sonnet', 'gemini-2.5-pro', 'gpt-4.1', 'llama-maverick']
}
```

## 🚀 Производительность

### Таймауты:
- **Per-model timeout**: 50 секунд
- **Selector timeout**: 12 секунд
- **Total timeout**: зависит от медленнейшей модели

### Результаты в логах:
```
✅ [chatId] Субагент claude-sonnet: 2543 символов за 3.2с
✅ [chatId] Субагент gemini-2.5-flash: 1890 символов за 2.1с
🏆 [chatId] Лучший: claude-sonnet из 2 (3.2с)
```

